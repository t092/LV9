# 設定與更新 UI 顏色變數 (CSS)

如果需要為新章節新增主題色，或修改現有的 UI 主視覺，請遵循以下規範：

1. **集中於 `:root` 定義**:
   - 所有的顏色都必須定義在 `style.css` 開頭的 `:root` 偽類別中作為 CSS 自訂屬性 (Custom Properties)。
   - 不要在個別 CSS 選擇器中直接寫死 Hex 色碼或 rgb 色碼。

2. **命名慣例**:
   - 顏色命名應具有語意，統一加上 `--` 前綴。
   - 色系應以階層數字表示深淺：例如 `--gold-100` (淺色背景用) 延伸至 `--gold-600` (深色文字用)。
   - 情境色應明確：如 `--success`、`--error`、`--primary-color` 等。

3. **應用於新章節**:
   - 當為主畫面或章節卡片引入新顏色時，可建立一組新的 `--{主題名}-{100~600}`。
   - 隨後針對組件類別（如 `.chapter-card` 的 `border-color` 屬性）套用 `var(--{新增的變數名})`，以保持擴充性與設計一致性。
